# Чек-лист подключения первого салона (пилот)

Используйте этот список при onboarding первого реального салона. Каждый пункт — обязательный.

## Перед началом (разработчик / DevOps)

- [ ] Supabase: применены миграции, включая `20250622000004_salons_and_integrations.sql`
- [ ] Сервер развёрнут публично (Docker / Render / VPS), не только localhost
- [ ] Переменные окружения на сервере: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `OPENROUTER_API_KEY`
- [ ] `APP_URL` = публичный URL приложения
- [ ] `TELEGRAM_CHAT_ID` = Telegram ID владельца/админа (для уведомлений)
- [ ] Проверка деплоя: `DEPLOY_URL=https://ваш-домен node scripts/verify-deployment.mjs`

## 1. Подготовка каталога салона

- [ ] В разделе **Услуги** — добавлены все услуги с ценой и длительностью
- [ ] В разделе **Сотрудники** — добавлены мастера (specialties помогают боту выбрать мастера)
- [ ] В разделе **Клиенты** — можно оставить пустым (клиенты создаются из Telegram)

## 2. Создание Telegram-бота

- [ ] Открыть [@BotFather](https://t.me/BotFather) в Telegram
- [ ] Команда `/newbot` → задать имя и username (например `My Salon Bot`, `@mysalon_book_bot`)
- [ ] Скопировать **Bot Token** (формат `123456789:AA...`)
- [ ] *(Рекомендуется)* `/setdescription` — описание для клиентов
- [ ] *(Рекомендуется)* `/setabouttext` — кратко о салоне

## 3. Подключение через Integrations

- [ ] Войти в панель салона → **Integrations** (боковое меню)
- [ ] Нажать **Connect Telegram** / **Подключить Telegram**
- [ ] Вставить Bot Token → **Connect**
- [ ] Статус: **Connected**, отображается `@username` бота
- [ ] Нажать **Open bot in Telegram** — бот открывается
- [ ] *(Опционально)* **Send test notification** — сообщение приходит админу

## 4. Проверка статуса

- [ ] На странице Integrations: зелёный статус Connected
- [ ] `GET /api/telegram/status` → `{ connected: true, bot: "..." }`
- [ ] После перезапуска сервера статус остаётся Connected (токен сохранён в БД)

## 5. Тест записи через Telegram

- [ ] Открыть бота от имени тестового клиента (не админ)
- [ ] Отправить любое сообщение → бот отвечает и показывает услуги из каталога
- [ ] Пройти сценарий: услуга → дата → время → имя → телефон
- [ ] Бот подтверждает запись
- [ ] Админ получает уведомление в Telegram (если настроен `TELEGRAM_CHAT_ID`)
- [ ] Проверка синхронизации:
  ```bash
  API_BASE=https://ваш-домен/api node scripts/verify-telegram-sync.mjs --phone +7XXXXXXXXXX
  ```
- [ ] **Bookings** — запись со статусом `scheduled`
- [ ] **Calendar** — запись на выбранную дату
- [ ] **Dashboard** — запись в «сегодня» (если дата сегодня)
- [ ] **Reminders** — напоминание `pending`
- [ ] **Statistics** — данные загружаются без ошибок

## 6. Тест подтверждения (админ)

- [ ] В **Bookings** → подтвердить запись (`confirmed`)
- [ ] Статус обновился в Bookings и Calendar

## 7. Тест переноса через Telegram

- [ ] В боте написать «перенести запись» / «перенос»
- [ ] Ввести телефон тестового клиента
- [ ] Выбрать новую дату и время
- [ ] Bookings / Calendar / Reminders обновились

## 8. Тест отмены через Telegram

- [ ] В боте написать «отменить запись» / «отмена»
- [ ] Подтвердить отмену
- [ ] Bookings: статус `cancelled`
- [ ] Calendar: запись не в активном списке
- [ ] Reminders: напоминание `failed` (не `pending`)

## 9. Тест AI-ответов

- [ ] Задать боту общий вопрос о салоне — ответ на языке клиента
- [ ] Бот не повторяет уже собранные данные
- [ ] При отсутствии `OPENROUTER_API_KEY` — сообщить разработчику (бот не ответит)

## 10. Финальная проверка

- [ ] `npm run build` проходит без ошибок
- [ ] Владелец салона может повторить шаги 2–8 **без участия разработчика**
- [ ] Инструкция для владельца передана: `docs/SALON_OWNER_GUIDE.md`

---

**Известные ограничения пилота (не блокируют старт, но важно знать):**

- Напоминания создаются в системе, но **автоматическая отправка email/SMS пока не реализована**
- Один сервер = один бот = один салон
- Нет разделения доступа между пользователями (auth — следующий этап)
